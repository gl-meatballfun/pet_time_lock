# 宠物时间锁 — 最终开发方案（v1.0）

## 1. 项目目标

打造一款基于 **本地大模型** 驱动的成长型电子宠物应用，陪伴中小学生成长：

1. **情绪价值**：宠物能根据学生状态、学习情况、互动历史，生成温暖、鼓励、陪伴式对话。
2. **学业保持与提高**：通过动态任务、个性化问题、错题巩固，帮助学生保持学习习惯并针对性提升。
3. **手机沉迷防范**：统计手机使用时长，由宠物主动进行温和提醒，配合专注模式培养自律。

## 2. 核心设计决策

| 决策项 | 最终方案 |
|---|---|
| 货币体系 | **单一主币 + 学科积分**：成长币（通用）+ 文科积分 / 理科积分 / 健康积分 |
| 答题限时 | 以**推荐用时**为主；背诵、填空类考察可增加**限时机制** |
| 大模型 | **本地模型**为主，低性能设备 fallback 到本地模板 |
| LLM 使用比例 | 大比例走本地 LLM，关键场景（问候、进化、答错安慰、阶段转换）优先 |
| 隐私策略 | 不上传具体错题内容，仅向 LLM 传递**年级、宠物阶段、薄弱学科类型** |
| 配置安全 | API Key / 模型配置等敏感信息**加密存储**（AES） |
| 防沉迷一期 | 仅做**使用时长统计 + 宠物温柔提醒**，不做原生强制限制 |
| 家长端 | **二期**引入家长周报 / 使用报告 |
| 启动方式 | 从**一期本地核心闭环**开始，暂不接入 LLM，验证核心玩法后再赋能 |

## 4. 一期：本地核心闭环（6–8 周）

### 4.1 货币与积分体系

#### 单一主币：成长币
- 用于宠物阶段成长、解锁阶段功能、购买高级商品。
- 产出：完成每日任务、阶段跃迁、成就。

#### 学科积分
| 积分类型 | 产出学科 | 主要用途 |
|---|---|---|
| 文科积分 | 语文、英语 | 购买提升快乐 / 知识的商品 |
| 理科积分 | 数学、物理 | 购买恢复饥饿 / 健康的食物 |
| 健康积分 | 专注模式、完成护理任务 | 购买药品、恢复类商品 |

#### 数值平衡公式（按年级梯度）
```
基础奖励 = 5 + (grade - 1) / 2   // 小学低年级约 5，高中约 10
随机加成 = 0 ~ 3
最终奖励 = 基础奖励 + 随机加成
```

### 4.2 宠物生长阶段

| 阶段 | 时长 | 解锁条件 | 内容难度 | 互动能力 |
|---|---|---|---|---|
| 蛋孵化期 | 3–9 天 | 创建宠物 | 略低于年级 | 简单情绪、emoji |
| 幼崽期 | ≥ 1 个月 | 时间达标或 growthXp 达标 | 与年级持平 + 30% 日常知识 | 简短语言、求互动 |
| 培育期 | 约 6 个月 | 时间 + xp 双达标 | 与年级持平 + 20% 日常知识 | 讲解知识点 |
| 成熟期 | 持续 | 进入后不回退 | 高于年级 | 导师式陪伴、主动提醒 |

**阶段加速机制**：每学习 1 小时可减少 0.5 天孵化/成长时间。

**阶段跃迁奖励**：一次性奖励成长币 + 专属装饰 + 新外观。

### 4.3 商店与背包

#### 商品分类
- **食物**：恢复饥饿，少量恢复快乐/健康
- **玩具**：提升快乐，可能消耗饥饿/健康
- **药品**：恢复健康
- **装饰**：永久解锁外观

#### 一期商品示例
| 商品 | 分类 | 价格 | 效果 |
|---|---|---|---|
| 小饼干 | 食物 | 10 理科积分 | 饥饿 +15，快乐 +5 |
| 豪华大餐 | 食物 | 30 理科积分 | 饥饿 +30，快乐 +10，健康 +5 |
| 知识蛋糕 | 食物 | 10 文科积分 + 20 理科积分 | 饥饿 +20，快乐 +15，知识 +10 |
| 弹力球 | 玩具 | 15 文科积分 | 快乐 +20，饥饿 -5 |
| 草药 | 药品 | 10 健康积分 | 健康 +20 |
| 学霸帽 | 装饰 | 30 文科积分 + 30 理科积分 | 解锁外观 |

### 4.4 本地题库与答题

- 按年级、学科分类，一期至少 **200–500 题**。
- 题型支持：选择题、填空题、背诵题。
- 背诵/填空题可显示限时，其他题型显示推荐用时。
- 答对后根据学科奖励对应积分，答错不扣分但记录错题。

### 4.5 每日任务（本地模板）

每天生成 3–5 个任务：
1. 文科任务（答语文/英语题）
2. 理科任务（答数学/物理题）
3. 护理任务（喂食/玩耍）
4. 专注任务（完成一次专注模式）
5. 防沉迷任务（减少娱乐 App 使用时长）

### 4.6 防沉迷基础版

- 统计每日手机使用时长（沿用现有 UsageStatsManager / Local mock）。
- 超过阈值时宠物主动温柔提醒：
  - 1 小时："休息一下吧，眼睛累了"
  - 2 小时："今天用手机好久啦，陪我学道题放松一下？"
- 专注模式：计时 + 宠物陪伴 + 完成奖励。

### 4.7 数据库设计（v3）

新增/调整表：
- `pet_state`：增加成长币、文科积分、理科积分、健康积分字段
- `shop_items`：商品 catalog
- `inventory`：用户背包
- `daily_tasks`：每日任务
- `reward_logs`：积分流水
- `wrong_answers`：错题本（内容 ID、学科、次数、最近错误时间）
- `educational_content`：本地题库，增加题型、推荐用时、限时字段

### 4.8 关键 UI

- 主界面：宠物 + 四维属性 + 三种积分余额 + 商店/任务入口
- 商店页：分类 Tab、商品详情、购买按钮
- 背包页：物品列表、使用/装备
- 任务页：今日任务、进度、领取奖励
- 专注页：计时、宠物陪伴、返回按钮

## 5. 二期：本地 LLM 赋能（6–8 周）

### 5.1 本地大模型接入

#### 推荐方案：llama.cpp + Flutter FFI
- 模型选择：Qwen2.5-1.8B / Phi-3-mini / Llama-3.2-1B
- 量化：Q4_K_M 或 Q5_K_M，平衡体积与效果
- 设备要求：中高端手机（6GB 运存以上）
- 低端机：自动 fallback 到本地模板

#### 模型管理
- 首次启动引导下载模型（分发型号可选）。
- 模型文件存储在应用私有目录。
- 提供"测试模型"按钮，验证本地生成是否正常。

### 5.2 LLM 应用场景

| 场景 | 输入上下文 | 输出 |
|---|---|---|
| 互动回复 | 宠物阶段、属性、主人年级、今日学习情况、互动历史 | 文案 + 情绪标签 + 动作指令 |
| 任务生成 | 阶段、薄弱学科、宠物状态、今日已做任务 | 3–5 个个性化任务 |
| 问题生成 | 年级、学科、阶段、日常知识比例 | 题目 + 选项 + 答案 + 解析 |
| 错题讲解 | 薄弱知识点、年级 | 讲解文案 + 鼓励话术 |

### 5.3 情绪价值强化

- **安全护栏**：本地敏感词过滤 + 价值观审核。
- **个性化记忆**：记录学生偏好科目、常见错题类型、昵称等。
- **动作指令**：LLM 输出 `action` 字段（如 `wag_tail`、`jump`、`think`），宠物做对应动画。
- **防沉迷情绪提醒**：检测到使用娱乐 App 超时时，生成温柔提醒而非说教。

### 5.4 内容安全策略

1. System Prompt 强制约束：温暖、鼓励、适龄、健康积极。
2. 本地校验：敏感词、JSON 合法性、选项完整性。
3. 违规内容 fallback：替换为 "抱歉，我暂时想不到合适的回复😜"。
4. 不上传具体错题，上下文脱敏。

### 5.5 家长端

- 学习周报：答题数、正确率、专注时长、知识点分布。
- 手机使用报告：每日使用时长、娱乐 App 占比。
- 宠物成长报告：阶段、属性变化、获得成就。

## 6. 三期：防沉迷深化（4–6 周）

- 原生屏幕时间权限（安卓 AccessibilityService / iOS Screen Time API）。
- 应用限额强制提醒与软阻断。
- 家长端实时查看与远程提醒。
- 成长报告导出与分享。

## 7. 关键风险与应对

| 风险 | 影响 | 应对 |
|---|---|---|
| 本地模型性能不足 | 低端机生成慢、发热 | 低端机 fallback 本地模板；提供模型大小可选 |
| 模型包体过大 | App 体积几百 MB | 首次启动按需下载；基础功能不依赖模型 |
| LLM 输出不合规 | 未成年人安全 | System Prompt + 本地敏感词 + 人工审核样本 |
| 积分通胀/不足 | 经济系统崩坏 | 建立数值公式，开发期持续监控产出/消耗比 |
| 隐私泄露 | 学生数据上传第三方 | 纯本地模型 + 上下文脱敏 + API Key 加密 |
| 家长信任度 | 担心孩子过度依赖虚拟宠物 | 明确"辅助陪伴"定位，避免替代现实关系 |

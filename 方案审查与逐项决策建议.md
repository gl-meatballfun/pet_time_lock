# 方案审查与逐项决策建议

## 核心目标对齐性

先明确核心目标：**大模型驱动的成长型电子宠物，覆盖情绪陪伴、学业提分、防手机沉迷**，方案整体方向匹配，但需针对 “落地可行性、核心目标强化、风险兜底” 补充细节，以下按模块拆解并给出 “保留 / 调整 / 补充” 建议，便于逐项决策。

## 整体决策结论

### 1\. 能否进入开发阶段？

**可以进入开发阶段，但需先落地 “核心基础模块”，再迭代 LLM 模块**，建议分两期：

- 一期（基础可用）：分值体系 \+ 商店 \+ 宠物阶段（纯本地逻辑）\+ 基础 UI 集成，验证核心流程（答题得币、购物、阶段成长）；

- 二期（LLM 赋能）：集成大模型，实现动态回复 / 任务 / 问题，强化情绪价值和防沉迷。

### 2\. 开发优先级建议（逐项决策）

|模块|优先级|开发要点|
|---|---|---|
|分值体系 \+ 数据库|P0（必须）|先定义数值平衡公式，确保数据一致性|
|宠物阶段判定 \+ 基础 UI|P0（必须）|先落地时间 \+ xp 双维度判定，基础外观 / 对话|
|商店 \+ 背包 \+ 每日任务（本地模板）|P1（重要）|先落地基础商品 / 任务，LLM 后续迭代|
|LLM 服务封装 \+ 动态互动回复|P1（重要）|先落地基础调用 \+ 情绪回复，防沉迷逻辑同步加|
|动态任务 / 问题生成|P2（次重要）|一期用本地模板，二期替换为 LLM 生成|
|防沉迷关联功能|P1（重要）|一期先落地基础专注模式 \+ 提醒，二期强化 LLM 情绪提醒|

### 3\. 风险兜底建议

1. **LLM 成本风险**：先对接开源轻量模型（如 Llama2\-7B）做内测，验证效果后再对接商用模型；

2. **内容安全风险**：上线前做全量 prompt / 输出审核，避免违规内容；

3. **数值平衡风险**：开发阶段做灰度测试，监控货币产出 / 消耗比例，避免 “通货膨胀” 或 “奖励不足”；

4. **用户体验风险**：优先做小范围用户测试（10\-20 名学生），验证情绪陪伴、防沉迷提醒的接受度。

## 补充建议（核心目标强化）

1. **情绪价值落地细节**：

    - 给宠物增加 “记忆库”（存储用户的生日、喜欢的科目、错题类型），LLM 生成时调用，强化 “专属陪伴”；

    - 宠物回复避免 “机器感”，加入口语化语气词（如 “呀、啦、呢”），不同阶段语气差异（幼崽期多用叠词，成熟期多用鼓励性短句）。

2. **学业提分落地细节**：

    - 动态问题生成时关联 “教材版本”（如人教版 / 苏教版），避免脱离用户学习范围；

    - 宠物回复加入 “知识点拆解”（如答对数学题后，成熟期宠物说 “这个知识点是勾股定理，记住 a²\+b²=c² 哦～”）。

3. **防沉迷落地细节**：

    - 接入手机 “使用时长统计”（安卓 AccessibilityService / 苹果 Screen Time API），宠物主动提醒 “今天刷短视频已经 1\.5 小时啦，要不要和我一起答两道题放松一下？”；

    - 专注模式完成后，宠物生成 “专注证书”（可分享），强化正向激励。

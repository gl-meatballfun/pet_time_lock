 Android 桌面悬浮宠物原型 — 实现计划

 Context

 用户希望「宠物时间锁」应用具备传播性，核心卖点是宠物能跳出 App 界面，在手机桌面独立存在。经过技术可行性分析：

 - 微信小程序无法承载屏幕时间监控、悬浮窗、后台守护等核心能力，建议 App 为主、小程序后续仅作引流。
 - Android 可通过 SYSTEM_ALERT_WINDOW 实现全局悬浮宠物；iOS 无法做到真正的跨应用悬浮窗，只能降级为 Widget / Live Activity / 通知。

 本计划聚焦：在现有 Flutter 项目上实现一个 Android 全局悬浮宠物原型，让它既能在用户开启后常驻桌面，也能在关键节点（专注完成、使用超时、宠物进化）自动「跳出来」。

 Recommended Approach

 使用 flutter_overlay_window 插件作为悬浮窗引擎，复用现有 AnimatedPetWidget 的视觉逻辑，通过数据库作为 overlay 与主 App 之间的状态同步源。

 为什么选 flutter_overlay_window

 - 成熟封装了 Android WindowManager + TYPE_APPLICATION_OVERLAY + 第二 FlutterEngine 的复杂度。
 - 支持拖拽、显示/隐藏、大小调整、跨引擎数据传递。
 - 相比纯原生手写 overlay service，开发量小得多，适合快速验证原型。

 状态同步策略

 Overlay 运行在独立 Flutter 引擎，无法共享 PetCubit。因此：

 - 单一数据源：DatabaseHelper 的 SQLite 文件。
 - Overlay 启动时从 DB 读取 PetState；主 App 在宠物状态变化后通过 FlutterOverlayWindow.shareData 发送 refresh_pet 让 overlay 重新读取。
 - 复杂交互（喂食/玩耍等）由 overlay 通过 MethodChannel 拉起主 App，并附带 payload，主 App 恢复后执行对应操作。

 UI/UX 设计

 折叠态（默认）

 - 大小：72×72 dp 圆形。
 - 视觉：与 AnimatedPetWidget 一致，使用当前 stage 表情 + 心情色背景 + 轻微上下浮动。
 - 行为：拖拽移动；点击展开菜单。
 - 位置：记住上次拖拽位置（SharedPreferences）。

 展开态

 - 在宠物上方弹出圆角白卡菜单。
 - 操作项：
   - 喂食（绿）
   - 玩耍（粉）
   - 学习（青）
   - 专注（蓝）
   - 回应用（橙）
 - 点击任一操作后：菜单收起，拉起主 App 并携带对应 payload。
 - 5 秒无操作自动收起。

 触发态（专注完成 / 超时 / 进化）

 - 临时放大到 120×120 dp，显示对应表情与气泡文案。
 - 8 秒后自动恢复，或点击后进入展开菜单。

 Payload 枚举（lib/models/overlay_payload.dart）

 enum OverlayPayload {
   feed('feed'),
   play('play'),
   pet('pet'),
   learn('learn'),
   focus('focus'),
   openApp('open_app'),
   overLimit('over_limit'),
   focusComplete('focus_complete'),
   evolution('evolution');

   final String value;
   const OverlayPayload(this.value);
 }

 Trigger Integration Points

 1. 专注完成：FocusScreen._completeFocus() → OverlayService.showOverlayWithTrigger(focusComplete, '专注完成啦！')
 2. 应用超时：ScreenTimeService 检测限额后 → OverlayService.showOverlayWithTrigger(overLimit, '你用手机有点久啦...')
 3. 宠物进化：PetCubit._applyAndEmit() 检测到 stage 变化 → OverlayService.showOverlayWithTrigger(evolution, '我进化啦！')

 Verification Steps

 1. 安装后首次开启悬浮宠物 → 跳转系统悬浮窗权限页 → 授权后宠物出现。
 2. 返回桌面 / 打开其他 App → 宠物仍可见并可拖拽。
 3. 点击宠物展开菜单 → 点击「喂食」→ 主 App 被拉起并执行喂食。
 4. 主 App 内喂食后 → overlay 宠物颜色/表情同步更新。
 5. 设置 1 分钟专注 → 完成后 overlay 自动弹出庆祝状态。
 6. iOS 构建通过，设置页显示「iOS 暂不支持悬浮窗」。
 7. Android Studio Profiler 观察 30 分钟，overlay 内存无明显泄漏。

 │         Risk         │                            Mitigation                            │

 Out of Scope for This Prototype

 - iOS Widget / Live Activity（后续迭代）
 - 开机自启恢复 overlay
 - 夜间自动隐藏 / 休眠时段
 - 悬浮窗内直接完成喂食等写操作（仅拉起主 App）
